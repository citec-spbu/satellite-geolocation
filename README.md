# drone-localization

## Описание проекта

**Drone Localization** — это сервис для определения географических координат аэрофотоснимков (с дронов или высотных камер) путём их сопоставления со спутниковой картой.
Проект решает задачу cross-view geo-localization: на вход подаётся изображение «с воздуха», а на выходе сервис возвращает точные GPS-координаты (широта/долгота), оценку уверенности модели и соответствующий спутниковый тайл для визуализации.

## Пайплайн обработки

Обработка изображений проходит в два основных этапа: **поиск (retrieval) + уточнение (refinement)**.
1. В систему загружается галерея спутниковых снимков, с которыми будут сопоставляться тестовые фото с дрона
2. Первая модель извлекает из них эмбеддинги, они помещаются в векторную базу данных (Qdrant)
3. Retrieval сервис осуществляет поиск похожих и возвращает соответствующее фото со спутника
4. Пара фото (с дрона и со спутника) одного и того же объекта поступают во вторую модель
5. Refinement сервис возвращает координаты дрона и точку на фото со спутника 

## Используемые модели

### Retrieval (поиск)
**Задача:** сопоставление снимка с дрона спутниковому снимку той же местности

**Архитектура:** two_view_net с ConvNeXt-Tiny backbone и Triplet Attention

**Как работает:**
1. Набор изображений со спутника пропускается через backbone, формируется галерея эмбеддингов
2. Извлекается эмбеддинг для тестового изображения с дрона
3. Поиск соответвующего здания по косинусной близоти

Модель обучалась на датасете SUES-200 (200 локаций, снимки со спутника и дрона 256×256) в течение 152 эпох. Функция потерь: CrossEntropy + TripletLoss. Оптимизатор — SGD с lr=0.01.

### Refinement (уточнение)
Задача: определение координат дрона на снимке

Архитектура: MixFormer Convolutional-Vision-Transformer backbone, FPN(Feature Pyramid Network) neck и MLP head

Как работает:
1. Изображения со спутника и дрона одновременно передаются в backbone
2. На выходе получаем heatmap положения дрона на снимке со спутника
3. После преобразований получаем наиболее вероятное положение дрона на снимке

Модель обучалась на датасете Ul14 (6,768 фотографий со спутника и дрона и использованием аугментации данных) в течение 120 эпох. Функция потерь: Balanced CrossEntropy Loss. Оптимизатор — AdamW с MultiStepLR scheduler.

## Технологический стек

|Компонент|Технологии|
|-------|-------------|
|Backend|FastAPI|
|Frontend|Streamlit|
|ML|Pytorch, TorchVision|
|БД эмбеддингов|Qdrant|
|Хранение изображений|MinIO|
|Контейнеризация|Docker|
|Управление зависимостями|Pixi|

## Установка и запуск (TODO)

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/citec-spbu/satellite-geolocation
   cd satellite-geolocation
   ```
2. **Подготовьте веса моделей:**

   Поместите файлы весов для моделей в директорию `weights/`:
   - `retrieval.pth` — веса для модели поиска
   - `net_best.pth` — веса для модели уточнения
3. **Запустите сервисы:**
   ```bash
   docker-compose up --build
   ```

4. **Доступ к сервисам:**
   - **Streamlit UI**: http://localhost:8501
   - **FastAPI Backend**: http://localhost:8000
   - **Qdrant Dashboard**: http://localhost:6333
   - **MinIO Console**: http://localhost:9001 (логин/пароль: `minioadmin/minioadmin`)

5. **Остановка сервисов:**
   ```bash
   docker-compose down
   ```
